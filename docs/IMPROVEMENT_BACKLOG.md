# Wix 改进 Backlog

每日 Cloud Agent 挖掘与实现记录。优先级：**P0**（高影响/低复杂度或强场景需求）→ **P1** → **P2**。

## 已发现项

### P0

| 项 | 场景 | 当前不足 | 建议方案 |
| --- | --- | --- | --- |
| **睡眠定时 + 渐出停止** | 睡前播放白噪音，希望到时自动渐弱并停止，避免惊醒 | 无定时器；停止为即时切断 | 预设时长（15/30/45/60 分钟）、倒计时显示、到期前 30s 主音量线性渐出后暂停 |
| **混音状态持久化** | 用户刷新或误关页面后希望恢复上次组合与音量 | 仅主题、入门标记、自定义音轨持久化 | `localStorage` 保存 `MixerState` 快照，启动时合并默认项 |
| **主音量记忆** | 每次打开主音量回到 82% | 无用户偏好记忆 | 单独 key 存 `masterVolume`，与混音快照可合并 |

### P1

| 项 | 场景 | 当前不足 | 建议方案 |
| --- | --- | --- | --- |
| **场景预设（收藏组合）** | 专注/睡眠/旅行固定搭配 | 仅 8 种内置声，无命名预设 | 预设 CRUD + 一键加载到 `MixerState` |
| **Android 后台播放与锁屏控制** | 锁屏/切 App 继续听 | 纯 WebView，无 Media Session | Capacitor 插件或原生 foreground service |
| **PWA `start_url` 与 Pages 子路径** | GitHub Pages 子目录安装 PWA | manifest `start_url: '/'` 可能与 `VITE_BASE_PATH` 不一致 | 构建时注入 base 到 manifest |
| **音频加载失败重试** | 弱网或 OGG 缺失 | `fetch` 失败直接抛错 | 指数退避重试 + 用户可见错误态 |
| **减少动效（`prefers-reduced-motion`）** | 前庭敏感用户 | 无媒体查询适配 | CSS/JS 缩短或关闭过渡 |
| **底部抽屉焦点陷阱** | 键盘/读屏用户 | 有 `role="dialog"` 但无 focus trap | `focus-trap` 或自管 Tab 循环 |

### P2

| 项 | 场景 | 当前不足 | 建议方案 |
| --- | --- | --- | --- |
| **导出/分享混音配置** | 向朋友分享「雨+篝火」配方 | 无序列化 | JSON 导出/导入或 URL 编码 |

| **混音 ARIA 实时区域** | 读屏知播放/定时状态 | 部分控件有 label | `aria-live` 播报播放与定时 |

### 体验场景与缺口（摘要）

| 场景 | 典型需求 | 现状 |
| --- | --- | --- |
| 睡眠 | 定时、渐出、低亮度 UI | 无定时；主题可深色 |
| 专注 | 预设、快速恢复 | 需每次点选声卡 |
| 冥想 | 慢速播放、简单组合 | 有全局/ per-track 速度 |
| 哄娃 | 长时间稳定循环 | 可混音，无定时渐出 |
| 屏蔽噪音 | 粉/棕噪 + 雨声 | 内置齐全 |
| 旅行/办公 | 离线 PWA、小体积 | PWA 缓存静态资源；内置 OGG 需构建时存在 |
| 自定义内容 | 导入本地音频 | IndexedDB 已实现 |

## 本次选中项

**睡眠定时 + 渐出停止（P0）**

- **理由**：睡眠/哄娃/午休是白噪音核心场景；实现边界清晰（定时状态 + 渐出 + UI + 测试），单 PR 可交付用户可感知能力；不依赖后端与 Android 原生改造。
- **范围**：预设 15/30/45/60 分钟、取消、抽屉配置 + dock 倒计时、到期 30 秒主音量渐出后自动暂停并恢复主音量设定值。
